**Step 1: Create `src/components/Providers.tsx`** (new file):
```typescript
"use client";

import { MotionConfig } from "motion/react";
import { NuqsAdapter } from "nuqs/adapters/next/app";

export function Providers({ children }: { children: React.ReactNode }) {
  return (
    <MotionConfig reducedMotion="user">
      <NuqsAdapter>{children}</NuqsAdapter>
    </MotionConfig>
  );
}
```

This wraps both `MotionConfig` (for reduced-motion) and `NuqsAdapter` (for URL state) in a single client component. `MotionConfig` uses React Context, which requires `"use client"`. The `reducedMotion="user"` prop tells Motion to check the OS `prefers-reduced-motion` setting and automatically suppress transform/layout animations when enabled.

**Step 2: Update `src/app/layout.tsx`**:
- Remove the `NuqsAdapter` import: `import { NuqsAdapter } from "nuqs/adapters/next/app";`
- Add the Providers import: `import { Providers } from "@/components/Providers";`
- Replace `<NuqsAdapter>{children}</NuqsAdapter>` with `<Providers>{children}</Providers>`

The root layout remains a Server Component. Only the Providers wrapper is a client component.

**Step 3: Add CSS reduced-motion rule to `src/app/globals.css`**:
Add the following at the end of the file (after the `@layer utilities` block):

```css
@media (prefers-reduced-motion: reduce) {
  .animate-pulse {
    animation: none;
  }
  .animate-spin {
    animation: none;
  }
}
```

This handles CSS-based animations that `MotionConfig` does not control (skeleton pulse, loader spin). This is a global rule that applies to all elements using these Tailwind animation classes. It complements the per-component Motion approach.

Do NOT add `motion-reduce:animate-none` to individual components -- the global CSS rule is more maintainable and catches all current and future uses of these classes.
