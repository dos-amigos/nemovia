2. Check enrich_logs for recent successful enrichment runs:
   ```
   curl -s "https://lswkpaakfjtxeroutjsb.supabase.co/rest/v1/enrich_logs?order=completed_at.desc&limit=3&select=geocoded_count,geocode_failed,llm_count,error_message,completed_at" \
     -H "apikey: {NEXT_PUBLIC_SUPABASE_ANON_KEY}" \
     -H "Authorization: Bearer {SUPABASE_SERVICE_ROLE_KEY}"
   ```
   Expect: at least one row with geocoded_count > 0 and error_message is null.

3. Check sagre table for enriched rows with valid coordinates:
   ```
   curl -s "https://lswkpaakfjtxeroutjsb.supabase.co/rest/v1/sagre?status=eq.enriched&location=not.is.null&select=id,title,location_text,status,food_tags,feature_tags,enhanced_description,source_url&limit=5" \
     -H "apikey: {NEXT_PUBLIC_SUPABASE_ANON_KEY}" \
     -H "Authorization: Bearer {SUPABASE_SERVICE_ROLE_KEY}"
   ```
   Expect: rows with non-null title, location_text, food_tags (array with items), enhanced_description, and source_url containing "eventiesagre".

4. Verify coordinates are valid (not null, not 0,0) by checking a sagra's location via PostGIS:
   ```
   curl -s "https://lswkpaakfjtxeroutjsb.supabase.co/rest/v1/rpc/find_nearby_sagre" \
     -X POST \
     -H "apikey: {NEXT_PUBLIC_SUPABASE_ANON_KEY}" \
     -H "Authorization: Bearer {SUPABASE_SERVICE_ROLE_KEY}" \
     -H "Content-Type: application/json" \
     -d '{"lat": 45.5, "lng": 11.5, "radius_km": 200, "max_results": 5}'
   ```
   Expect: non-empty array with sagre that have coordinates (the RPC only returns geolocated sagre).

5. Check for any sagre stuck in pending_geocode (should be 0 or decreasing):
   ```
   curl -s "https://lswkpaakfjtxeroutjsb.supabase.co/rest/v1/sagre?status=eq.pending_geocode&select=count" \

     -H "Authorization: Bearer {SUPABASE_SERVICE_ROLE_KEY}" \
     -H "Prefer: count=exact"
   ```

Read all API keys from .env file. Print results clearly showing pass/fail for each check.
