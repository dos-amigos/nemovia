If nuqs is not already installed (check package.json), also run `npm install nuqs`.

Then create 3 files:

1. **src/hooks/useGeolocation.ts** -- Client hook ("use client" is not needed for hook files, but the file will only be used in client components):
   - State interface: `{ lat: number | null, lng: number | null, error: string | null, loading: boolean }`.
   - Export `useGeolocation()` hook returning `{ lat, lng, error, loading, requestLocation }`.
   - `requestLocation` is a `useCallback` that checks `navigator.geolocation` support, sets loading=true, calls `getCurrentPosition` with options `{ enableHighAccuracy: false, timeout: 10000, maximumAge: 300000 }` (5 min cache).
   - On success: set lat/lng, clear error, loading=false.
   - On error: set error message, loading=false.
   - Do NOT call `getCurrentPosition` automatically on mount. Geolocation is opt-in only via `requestLocation`.

2. **src/components/search/SearchFilters.tsx** -- Client component ("use client"):
   - Import `useQueryStates`, `parseAsString`, `parseAsInteger`, `parseAsBoolean` from `nuqs`.
   - Import `useGeolocation` from `@/hooks/useGeolocation`.
   - Import `VENETO_PROVINCES` from `@/lib/constants/veneto`.
   - Import `FOOD_TAGS` from `@/lib/enrichment/llm`.
   - Import shadcn components: `Select, SelectContent, SelectItem, SelectTrigger, SelectValue` from `@/components/ui/select`, `Input` from `@/components/ui/input`, `Button` from `@/components/ui/button`.
   - Import `MapPin, X, Navigation` from `lucide-react`.

   Define nuqs filter parsers:
   ```typescript
   const filterParsers = {
     provincia: parseAsString,
     raggio: parseAsInteger.withDefault(30),
     cucina: parseAsString,
     gratis: parseAsBoolean,
     da: parseAsString,
     a: parseAsString,
     lat: parseAsString,  // stored as string in URL, parsed to number
     lng: parseAsString,
   };
   ```

   Component renders a filter panel with `space-y-3`:
   - **Geolocation button**: "Usa la mia posizione" with Navigation icon. On click, calls `requestLocation()`. When lat/lng available, updates the nuqs `lat` and `lng` params (as strings). Show loading spinner while requesting. Show "Posizione attiva" with green accent when active.
   - **Provincia**: shadcn `<Select>` with placeholder "Tutte le province", options from VENETO_PROVINCES (value = province name). Include a "Tutte" option that clears the filter (sets to null).
   - **Raggio km**: Only visible when geolocation is active (lat/lng in params). shadcn `<Input type="number">` with min=5, max=100, step=5, default 30. Label "Raggio km".
   - **Tipo cucina**: shadcn `<Select>` with placeholder "Tutti i tipi", options from FOOD_TAGS. Include a "Tutti" option that clears.
   - **Gratis toggle**: Two buttons ("Tutti", "Gratis") styled as a simple toggle. Use shadcn `Button` with variant "outline" for inactive and variant "default" for active.
   - **Date range**: Two `<Input type="date">` fields side by side. Label "Da" and "A". Values bound to `da` and `a` nuqs params.
   - **Reset button**: "Cancella filtri" with X icon. Clears all params by setting them all to null.

   All filter changes call `setFilters(...)` from `useQueryStates` with `{ shallow: false }` to trigger server re-render.

   IMPORTANT: When geolocation becomes available, set lat/lng as string params in the URL. The server component will parse these to pass to searchSagre.
