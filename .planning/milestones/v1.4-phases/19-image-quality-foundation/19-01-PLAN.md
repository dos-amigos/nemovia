       ```typescript

       ```

    3. Implement `runUnsplashPass(supabase: SupabaseClient): Promise<number>`:
       - Early return 0 if UNSPLASH_ACCESS_KEY is not set (graceful skip)
       - Query sagre WHERE image_url IS NULL AND is_active = true AND status IN ('enriched') ORDER BY created_at ASC LIMIT UNSPLASH_LIMIT
       - For each sagra: determine search query from food_tags[0] via TAG_QUERIES, fallback to DEFAULT_UNSPLASH_QUERY
       - Fetch `https://api.unsplash.com/search/photos?query={query}&orientation=landscape&per_page=10&content_filter=high` with Authorization header `Client-ID {UNSPLASH_ACCESS_KEY}` and `Accept-Version: v1`
       - Check `X-Ratelimit-Remaining` header: if below 5, stop early to preserve rate budget
       - On non-OK response (especially 403), break the loop immediately
       - Pick random photo from top 5 results for variety: `photos[Math.floor(Math.random() * Math.min(photos.length, 5))]`
       - Construct optimized URL: `${photo.urls.raw}&w=800&h=500&fit=crop&q=80`
       - Store credit as: `${photo.user.name}|${photo.user.links.html}?utm_source=nemovia&utm_medium=referral`
       - Update sagra: set image_url, image_credit, updated_at
       - Fire download tracking (Unsplash API requirement): `fetch(photo.links.download_location + '?client_id=' + UNSPLASH_ACCESS_KEY).catch(() => {})` -- fire and forget
       - Sleep UNSPLASH_SLEEP_MS between each API call
       - Return count of assigned images
