<objective>
Restructure the main layout from "contained by default" to "full-width by default" so that hero sections and future Netflix rows naturally span edge-to-edge without negative-margin breakout hacks, while content sections explicitly opt into max-w-7xl containment.

Purpose: The current layout applies `max-w-7xl px-4 sm:px-6 lg:px-8` on `<main>`, forcing every full-width element (hero, map, future Netflix rows in Phase 21) to use fragile negative-margin hacks. Inverting this -- full-width main, containment on content -- eliminates the hack pattern and prepares for Phase 21's 4-5 horizontal scroll rows.

Output: Updated layout.tsx, all page files adjusted to use explicit containment where needed, all negative-margin breakout hacks removed.
</objective>
