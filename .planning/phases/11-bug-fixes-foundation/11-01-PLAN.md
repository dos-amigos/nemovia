**BUG-04 fix** in `src/app/(main)/layout.tsx`:
Change line 10 from `max-w-lg` to `max-w-7xl`. The full class should be:
```
<main className="mx-auto max-w-7xl px-4 py-4">{children}</main>
```
Keep `px-4` padding intact. On mobile screens (< 1280px), `max-w-7xl` has no visual effect since the viewport is already narrower. On desktop, content expands to fill up to 1280px instead of being squeezed to 512px.

**BUG-03 fix** in `src/components/search/SearchFilters.tsx`:
The issue: when `filters.provincia` is `null`, the Select's `value` is `""` (empty string), which shows the placeholder text "Tutte le province" rather than the selected "Tutte" option. Users see placeholder styling and think no filter is active.

Fix: Change `value={filters.provincia ?? ""}` to `value={filters.provincia ?? "__all__"}`. This makes the Select show "Tutte" as the actively selected value when no provincia filter is applied, matching the `<SelectItem value="__all__">Tutte</SelectItem>` option.

The change is on line 112:
```typescript
value={filters.provincia ?? "__all__"}
```

This is safe because:
- When user selects "Tutte" (`__all__`), the `onValueChange` handler sets `provincia: null` (removes the filter)
- The query already returns all sagre when provincia is null (confirmed by research)
- The visual effect: "Tutte" appears as selected text instead of gray placeholder text
