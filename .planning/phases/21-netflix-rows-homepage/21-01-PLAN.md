**Deduplication logic (in-memory Set):**
Build rows sequentially, tracking shown IDs to prevent cross-row duplication:

1. `weekendSagre` -- first row, no dedup needed. Add all IDs to `shown: Set<string>`.

2. `gratisSagre` -- filter `allActive` where `is_free === true` AND `!shown.has(id)`. Take first 12. Add to `shown`.

3. `provinceSagre` -- determine `topProvinceName` from `provinceCounts[0]?.province`. Filter `allActive` where `province === topProvinceName` AND `!shown.has(id)`. Take first 12. Add to `shown`.

4. `foodSagre` -- count food_tags across all `allActive` (not just remaining). Find the most common tag with >= 3 remaining (non-shown) items. Filter `allActive` where `food_tags?.includes(topTag)` AND `!shown.has(id)`. Take first 12. The tag selection logic:
```typescript
const tagCounts = new Map<string, number>();
for (const s of allActive) {
  if (shown.has(s.id)) continue;
  for (const tag of s.food_tags ?? []) {
    tagCounts.set(tag, (tagCounts.get(tag) ?? 0) + 1);
  }
}
const topTag = [...tagCounts.entries()]
  .sort((a, b) => b[1] - a[1])
  .find(([, count]) => count >= 3)?.[0];
```
