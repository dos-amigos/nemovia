4. Expected: Deploy succeeds (green confirmation)

**Step 4: Invoke and verify**

1. Click "Invoke" (or Test button with body `{}`)
2. Expected: HTTP 200 response within 1-2 seconds with `{"status":"started","timestamp":"..."}`
3. Wait 90 seconds for background work to complete
4. Check results in SQL Editor:
   ```sql
   -- Check enrichment run log
   SELECT geocoded_count, geocode_failed, llm_count, error_message, duration_ms
   FROM enrich_logs ORDER BY run_at DESC LIMIT 5;

   -- Check enriched sagre
   SELECT title, location_text, status, food_tags, feature_tags,
          length(enhanced_description) AS desc_length
   FROM sagre WHERE status = 'enriched'
   ORDER BY updated_at DESC LIMIT 10;

   -- Verify description length constraint
   SELECT max(length(enhanced_description)) AS max_desc_chars
   FROM sagre WHERE status = 'enriched';

   -- Count enriched vs pending
   SELECT status, count(*) FROM sagre GROUP BY status ORDER BY status;
   ```

**Expected results:**
- `enrich_logs` has at least 1 row with `geocoded_count >= 0`
- `sagre` rows with `status = 'enriched'` show non-null `food_tags` (e.g., `{Pesce,Vino}`) and `enhanced_description`
- `max_desc_chars` result is 250 or less
- No `error_message` in enrich_logs (or error is explainable)

**If geocoded_count = 0 and llm_count = 0:** The scraper may not have run since Phase 2 setup. Invoke `scrape-sagre` first, wait 60s, then invoke `enrich-sagre` again.

**If Gemini returns 429 (quota exceeded):** The 250 RPD limit was hit during testing. Wait until midnight UTC and invoke again.
