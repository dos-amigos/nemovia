This file is the dynamic import target. Using `domMax` (not `domAnimation`) because BottomNav uses `layoutId` which is a layout animation feature only available in domMax (Pitfall 1 from RESEARCH.md).

**Step 2: Update Providers.tsx to wrap with LazyMotion.**

```tsx
"use client";

import { LazyMotion, MotionConfig } from "motion/react";
import { NuqsAdapter } from "nuqs/adapters/next/app";

const loadFeatures = () =>
  import("@/lib/motion-features").then((mod) => mod.default);

export function Providers({ children }: { children: React.ReactNode }) {
  return (
    <LazyMotion features={loadFeatures} strict>
      <MotionConfig reducedMotion="user">
        <NuqsAdapter>{children}</NuqsAdapter>
      </MotionConfig>
    </LazyMotion>
  );
}
```

Key points:
- `LazyMotion` wraps outside `MotionConfig` (both are providers from `motion/react`, not animated elements)
- `features={loadFeatures}` uses dynamic import for async loading -- initial bundle gets only the ~5KB LazyMotion shell
- `strict` prop is enabled to throw errors if any `motion.*` component is accidentally rendered inside the provider (catches missed migrations)
- `domMax` (~25KB) loads asynchronously on demand, supporting ALL motion features including layoutId
